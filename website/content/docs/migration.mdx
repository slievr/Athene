## From `@composio/agent-orchestrator` to `@slievr/athene`

The npm scope moved to `@slievr/athene`. If you installed under the old name:

```bash
npm uninstall -g @composio/agent-orchestrator
npm install -g @slievr/athene
```

Your config and data directory (`~/.agent-orchestrator`) don't change — no data migration needed. GitHub org and repo URLs (`slievr/Athene`) didn't change either.

## `athene spawn <project> <issue>` → `athene spawn <issue>`

The old two-argument form is rejected with an error. Pick the project via `-p` or by running from inside its worktree:

```bash
# old
athene spawn myproject 42

# new
athene spawn 42 -p myproject
# or, from inside the project's worktree:
athene spawn 42
```

## `ao init` → `athene start`

`ao init` has been removed. Use `athene start` instead; it auto-creates `agent-orchestrator.yaml` on first run and opens the dashboard in one step.

## Upgrading

Preferred:

```bash
athene update
```

The command detects your install method (`npm-global`, `pnpm-global`, git, or unknown) and picks the right upgrade path. Override behaviour with `--skip-smoke` or `--smoke-only` for git installs.

## Before major upgrades

1. `athene status` — see what's running.
2. `athene session cleanup` — archive finished sessions.
3. `athene stop --all` — halt every AO instance.
4. Upgrade.
5. `athene doctor` on the new version.
